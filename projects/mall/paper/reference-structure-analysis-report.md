## ✍️ 写作风格特征

### 1. 引用格式

- **图表引用**: "如图{X}-{Y}所示"
- **表格引用**: "表{X}-{Y}展示了"
- **代码引用**: "代码清单{X}-{Y}"

### 2. 内容组织方式

**典型章节结构**:
```
概念定义 → 详细说明 → 技术实现 → 应用场景
```

**段落特点**:
- 长度: 150-300字
- 风格: 详细解释型
- 技术深度: 深入到具体实现细节
- 示例: 包含配置和代码示例

### 3. 技术深度

- **描述级别**: Detailed (详细)
- **代码包含**: 是（配置文件和关键代码片段）
- **学术语气**: Formal (正式)

### 4. 各章节内容模式

| 章节 | 内容组织模式 |
|------|--------------|
| 绪论 | 背景分析 → 意义阐述 → 现状综述 → 目标设定 → 内容规划 |
| 技术基础 | 技术介绍 → 特点分析 → 应用场景 |
| 需求分析 | 可行性分析 → 功能需求 → 非功能需求 → 架构规划 |
| 详细设计 | 技术选型 → 模块设计 → 架构设计 → 详细设计 |
| 实现 | 环境配置 → 功能实现 → 关键代码 |
| 测试 | 测试环境 → 测试过程 → 测试结果 |
| 结语 | 成果总结 → 特色创新 → 问题不足 → 未来方向 |
