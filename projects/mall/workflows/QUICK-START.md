# 🚀 快速开始指南

## 📋 5分钟快速上手

### 第一步：准备工作
```bash
# 确保你在项目根目录
cd <projectRoot>

# 确保目录结构正确
# <projectRoot>/
# ├── source/          # 你的项目源码
# ├── templates/       # 大纲模板
# └── paper/           # 将自动创建
```

### 第二步：选择流程

#### 🎯 标准流程（推荐新手）
适合：第一次使用，没有参考论文

```bash
执行顺序：01 → 02 → 03 → 04(A/B/C) → 05 → 06 → 07 → 08 → 09
预计时间：2-3小时
```

#### 📚 参考论文流程（推荐有参考的用户）
适合：有参考论文，想要保持风格一致

```bash
# 1. 准备参考论文
将参考论文MD文件放入 reference-papers/ 目录

# 2. 执行流程
执行顺序：01 → 02 → 03 → 04(D) → 04.5(自动) → 05 → 06 → 07 → 08 → 09
预计时间：1.5-2小时（因为有参考可以更快）
```

### 第三步：执行工作流

#### 阶段一：项目分析（10-15分钟）
```
01. 源码真相扫描
    → 自动识别技术栈
    → 生成 source-truth.json

02. 实体类识别
    → 提取数据库实体
    → 生成三线表JSON

03. 单体ER图生成
    → 为每个实体生成ER图
    → 输出到 diagrams/er/
```

#### 阶段二：大纲与规划（15-20分钟）
```
04. 大纲确认
    → 选择模式A/B/C/D
    → 生成 outline.json

    [如果选择D] 04.5自动执行
    → 解析参考论文
    → 生成 reference-structure.json

05. 内容计划
    → 生成详细计划
    → 输出 content-plan.json

06. 章节切割
    → 切割为独立文件
    → 输出到 chapters/
```

#### 阶段三：内容生成（1-2小时）
```
07. 智能内容生成
    → 批量生成正文
    → 更新所有章节JSON

08. 资源提取
    → 提取图表清单
    → 分类资源类型

09. UML图表生成
    → 生成所有图表
    → 输出到 diagrams/uml/
```

## 🎯 关键决策点

### 决策点1：大纲模式选择（在步骤04）

| 模式 | 适用场景 | 准备工作 |
|------|---------|---------|
| **A** | 直接使用模板 | 无需准备 |
| **B** | 模板+AI建议 | 源码准备好 |
| **C** | 自定义大纲 | 准备大纲文本/JSON |
| **D** | 基于参考论文 | 准备参考论文MD |

### 决策点2：是否使用参考论文

#### ✅ 建议使用参考论文（模式D）如果：
- 有结构相似的参考论文
- 想要保持风格一致
- 追求更高的论文质量
- 节省规划时间

#### ⚠️ 不建议使用参考论文如果：
- 没有合适的参考论文
- 项目与参考差异太大
- 想要完全自定义结构

## 🔧 常见问题快速解答

### Q: 执行报错了怎么办？
1. 检查当前目录是否正确（项目根目录）
2. 检查上一步的输出文件是否存在
3. 查看具体工作流文档的"约束与容错"部分

### Q: 可以只执行某一步吗？
可以，但需要确保依赖的文件已存在。例如：
- 执行07需要：chapters/*.json存在
- 执行09需要：chapters/*.json和source-truth.json存在

### Q: 生成的内容不满意？
- **章节内容**：编辑 `chapters/*.json` 的 content 字段
- **图表**：编辑 `plantuml/*.puml`，重新执行09
- **大纲结构**：编辑 `outline.json`，重新执行05-09

### Q: 如何修改后重新生成？
```bash
# 修改了源码或实体类
重新执行：01 → 02 → 03

# 修改了大纲
重新执行：05 → 06 → 07 → 09

# 只修改了某些章节内容
重新执行：07（只处理修改的章节）
```

## 🎓 进阶使用

### 使用参考论文功能
```bash
# 1. 创建目录
mkdir -p reference-papers

# 2. 放入参考论文
cp /path/to/reference.md reference-papers/

# 3. 执行时选择模式D
# 系统会自动解析并应用参考规则
```

### 自定义配置
```bash
# 编辑路径配置
vim paper/path-config.json

# 编辑大纲模板
vim templates/outline.template.json

# 编辑内容模板
vim templates/content.template.json
```
