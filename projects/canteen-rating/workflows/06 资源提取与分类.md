# 📌 资源提取与分类

## 路径规范
projectRoot 取当前工作目录（pwd）的绝对路径

## 功能说明
- 使用 06-asset-extractor 工具从 content-plan.json 中提取包含图片和表格路径的内容块
- 按照资源类型自动分类存储为独立的 JSON 文件
- 生成的文件按原始路径结构保存到对应目录（pngs/ 和 tables/）

## 工具调用
使用 06-asset-extractor 工具提取资源相关内容：
- 输入文件：<projectRoot>/paper/content-plan.json
- 输出目录：<projectRoot>/paper/
- 自动分类到 pngs/ 和 tables/ 子目录
- 保留原始内容的完整信息

## 处理对象
### 图片资源块（imagePath）
- 包含 imagePath 字段的内容条目
- 自动提取文件名并保存为 JSON
- 示例：`activity-complaint-management.png` → `pngs/activity-complaint-management.json`

### 表格资源块（tablePath）
- 包含 tablePath 字段的内容条目
- 自动提取文件名并保存为 JSON
- 示例：`database-design.json` → `tables/database-design.json`

## 输出结构
```
<projectRoot>/paper/
├── pngs/                    # 图片相关内容块
│   ├── software-architecture.json
│   ├── activity-auth-management.json
│   ├── impl-rating-management.json
│   └── ...
└── tables/                  # 表格相关内容块
    ├── database-design.json
    ├── test-auth-management.json
    ├── performance-metrics.json
    └── ...
```

## 错误处理
- 如遇到 JSON 格式错误，先检查 content-plan.json 格式
- 确保输出目录有写入权限
- 验证提取的文件数量是否符合预期
- 同名文件会被覆盖，注意备份重要数据

## 预期输出
- 按资源类型分类的独立 JSON 文件
- 每个文件包含原始内容块的完整信息
- 统计报告显示处理的图片块和表格块数量
- 便于后续工具进一步处理资源相关内容

## 使用场景
- 论文章节按图表类型重新组织
- 为图表内容生成独立处理文件
- 资源相关内容的批量审核和管理
- 支持后续的图表内容自动化处理流程
