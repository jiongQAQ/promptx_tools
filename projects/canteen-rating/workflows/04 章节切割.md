# 📌 章节内容切割

## 路径规范
projectRoot 取当前工作目录（pwd）的绝对路径

## 功能说明
- 调用 json-chapter-splitter 工具对 content-plan.json 进行切割
- 根据 id 字段将包含 text、items 或 prompt 字段的条目分别保存为独立文件
- 输出到 chapter 目录，文件命名格式：chapter.{id}.json

## 工具调用
使用 json-chapter-splitter 工具切割内容计划文件：
- 输入文件：<projectRoot>/paper/content-plan.json
- 输出目录：<projectRoot>/paper/chapters
- 仅切割包含实际内容的条目（onlyWithContent: true）

## 错误处理
- 如遇到 JSON 格式错误，先修复 content-plan.json 后再次执行切割
- 确保输出文件保持原始内容完整性，不添加额外信息

## 预期输出
- 在 chapters 目录下生成多个 chapter.{id}.json 文件
- 每个文件包含对应条目的完整原始内容
