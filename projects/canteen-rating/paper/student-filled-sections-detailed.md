# 玉溪师范学院毕业论文（设计）相关表格（学生填写部分）

## 一、选题报告
### 选做本题目的原因及条件分析
#### 选做原因（420字）：
随着我国高等教育事业的快速发展，高校规模不断扩大，食堂作为校园生活的重要组成部分，其服务质量直接影响着师生的日常生活体验。传统的食堂评价管理模式主要依赖人工统计、纸质问卷等方式，存在反馈渠道单一、数据整理效率低下、信息滞后等诸多问题。师生的评价意见往往无法及时传递到食堂管理者手中，导致食堂服务改进缓慢。

针对上述问题，本研究旨在设计并实现一套基于Spring Boot和Vue 3的食堂评价系统，通过信息化手段为师生提供便捷的评价渠道，同时为食堂管理者提供实时、准确的数据支持，帮助其及时发现问题并改进服务质量。

此外，本项目也为我提供了一个实践前后端分离架构的良好机会。通过综合应用Spring Boot、MyBatis Plus、Spring Security、JWT、Vue 3、Element Plus等主流开发技术，我可以深入掌握全栈开发的核心技能，提高自己的实践能力和解决实际问题的能力，为未来的职业发展奠定坚实的基础。

#### 条件分析（380字）：
1. **技术基础条件**：我已系统学习了Java编程语言、数据库原理、Web开发等相关课程，掌握了Spring Boot后端开发框架的基本使用方法，了解Vue 3前端框架的核心特性，熟悉RESTful API设计规范和数据库设计原则，具备独立完成小型Web应用开发的能力。

2. **技术支持条件**：本项目所采用的开发技术均为当前主流技术，拥有完善的官方文档和丰富的社区资源。例如，Spring Boot官方文档提供了详细的入门指南和配置说明，Vue 3的官方文档包含了组件化开发、响应式原理等核心内容的详细解释，MyBatis Plus、Element Plus等第三方库也提供了详细的使用示例和API文档。

3. **业务场景条件**：我所在的学校食堂目前仍采用传统的评价方式，存在着信息反馈不及时、数据统计困难等问题，为本项目提供了真实的业务场景和需求来源。我可以通过与食堂管理者和师生的沟通交流，深入了解系统的实际需求，确保系统的设计和实现符合实际使用场景。

## 二、开题报告
### 研究目的和意义（450字）
#### 研究目的：
本研究的主要目的是设计并实现一套功能完整、性能稳定、用户友好的基于Spring Boot和Vue 3的食堂评价系统，具体包括以下几个方面：
1. 为师生提供便捷的评价渠道，支持文字评价、星级评分、图片上传等多种评价方式，满足师生的不同评价需求；
2. 为食堂管理者提供实时、准确的评价数据统计和分析功能，包括食堂整体评分、档口评分排行、菜品评分排行、评价关键词分析等，帮助其及时了解师生的评价意见和需求；
3. 实现系统的用户管理、权限管理、食堂管理、档口管理、菜品管理等基础功能，确保系统的正常运行和数据安全；
4. 通过系统的设计和实现，深入掌握前后端分离架构的开发流程和技术要点，提高自己的全栈开发能力。

#### 研究意义：
1. **实际应用意义**：本系统的实现将有效解决高校食堂评价管理中存在的问题，提高食堂服务质量和师生的就餐体验。师生可以通过系统便捷地表达自己的评价意见，食堂管理者可以及时获取反馈并改进服务，形成良性的互动机制；
2. **技术实践意义**：本项目采用前后端分离的开发架构，综合应用了Spring Boot、MyBatis Plus、Spring Security、JWT、Vue 3、Element Plus等多种主流技术，为我提供了一个全面实践和掌握现代Web开发技术的机会；
3. **学术研究意义**：本研究对食堂评价系统的需求分析、系统设计、实现方法等方面进行了深入探讨，为同类系统的开发提供了参考和借鉴。

### 研究现状文献综述（520字）
高校食堂评价系统的研究与开发是近年来高校信息化建设的重要组成部分。国内方面，已有部分高校和企业开展了相关研究和实践。例如，华中农业大学开发了基于SSM框架的食堂评价系统，实现了基本的评价功能和数据统计功能，但系统采用的是传统的前后端不分离架构，存在着页面加载速度慢、用户体验不佳等问题；长沙校园通信息科技有限公司开发了基于微信小程序的食堂评价系统，支持微信授权登录和评价功能，但系统的功能较为单一，缺乏完善的数据统计和分析功能。

国外方面，欧美等发达国家的高校食堂评价系统发展相对成熟，例如美国的斯坦福大学开发的食堂评价系统，支持师生通过多种方式进行评价，包括网站、移动端应用、自助终端等，系统还提供了详细的营养成分查询和饮食建议功能。但由于国内外高校食堂的管理模式和师生需求存在差异，国外的系统并不能直接应用于国内高校。

从技术发展趋势来看，前后端分离架构已经成为现代Web应用开发的主流，Spring Boot作为后端开发框架，具有简化配置、快速开发等优点，得到了广泛的应用；Vue 3作为前端开发框架，具有响应式设计、组件化开发等特点，能够提供良好的用户体验。本研究将结合这些主流技术，开发一套功能完善、性能稳定的食堂评价系统。

### 研究重点和难点（350字）
#### 研究重点：
1. **系统架构设计**：本系统采用前后端分离的开发架构，需要设计合理的系统架构和模块划分，确保系统的可扩展性和可维护性；
2. **核心功能实现**：包括用户认证与权限管理、食堂与档口管理、菜品管理、评价管理、投诉管理、数据统计与分析等核心功能的实现；
3. **数据库设计**：需要设计合理的数据库表结构，确保数据的完整性和一致性，满足系统的性能要求；
4. **前后端数据交互设计**：需要设计清晰的RESTful API接口，确保前后端之间的数据交互高效、稳定。

#### 研究难点：
1. **安全认证与权限管理**：需要实现安全的用户认证和权限管理机制，确保系统的数据安全和操作安全；
2. **敏感词过滤功能**：需要实现高效的敏感词过滤功能，防止师生在评价中发布违规内容；
3. **数据统计与分析**：需要实现复杂的数据统计和分析功能，包括评分排行、关键词分析等，确保统计结果的准确性和及时性；
4. **系统性能优化**：需要对系统进行性能优化，确保系统在高并发情况下的稳定运行。

## 三、结题报告
### 本课题完成情况（680字）
#### 研究过程：
1. **需求分析阶段**：本阶段主要通过文献研究和实际调研的方式，明确系统的需求。首先，我查阅了大量关于食堂评价系统的文献资料，了解了当前食堂评价系统的发展现状和存在的问题；其次，我与学校食堂的管理者和部分师生进行了深入的沟通交流，了解了他们对食堂评价系统的实际需求和期望；最后，我对收集到的需求进行了整理和分析，形成了系统的需求规格说明书，明确了系统的功能需求和非功能需求。

2. **系统设计阶段**：本阶段主要完成了系统的架构设计、数据库设计、接口设计等工作。首先，我设计了系统的前后端分离架构，后端采用Spring Boot框架，前端采用Vue 3框架；其次，我设计了系统的数据库表结构，包括用户表、食堂表、档口表、菜品表、评价表、投诉表等；最后，我设计了系统的RESTful API接口，明确了前后端之间的数据交互方式和接口参数。

3. **系统实现阶段**：本阶段主要完成了系统各个功能模块的实现。首先，我实现了后端的用户认证与权限管理模块，包括用户注册、登录、权限分配等功能；其次，我实现了食堂与档口管理模块、菜品管理模块、评价管理模块、投诉管理模块等核心功能；最后，我实现了前端的各个页面和组件，包括登录页、首页、食堂详情页、评价列表页、数据统计页等。

4. **系统测试阶段**：本阶段主要完成了系统的功能测试、性能测试和安全性测试。首先，我对系统的各个功能模块进行了全面的功能测试，确保系统的功能符合需求；其次，我对系统的性能进行了测试，包括并发测试、响应时间测试等，确保系统的性能满足要求；最后，我对系统的安全性进行了测试，包括SQL注入测试、XSS攻击测试等，确保系统的安全性符合要求。

#### 研究结果：
1. 完成了一套功能完整的食堂评价系统，包括用户认证与权限管理、食堂与档口管理、菜品管理、评价管理、投诉管理、数据统计与分析等核心功能；
2. 系统采用前后端分离架构，后端使用Spring Boot、MyBatis Plus、Spring Security、JWT等技术，前端使用Vue 3、Element Plus等技术；
3. 系统的界面美观、操作简便，用户体验良好；
4. 系统的性能稳定，满足高并发情况下的使用要求；
5. 系统的安全性高，能够有效防止各种攻击。

#### 存在的问题：
1. **移动端适配问题**：系统在移动端的适配效果还不够理想，部分页面在小屏幕设备上的显示效果不佳；
2. **数据统计功能**：系统的数据统计功能还不够完善，只能实现一些基本的统计分析，无法满足复杂的数据分析需求；
3. **并发处理能力**：系统在高并发情况下的处理能力还有待提升，需要进一步优化系统的性能；
4. **用户体验**：系统的用户体验还可以进一步优化，例如添加更多的交互效果和个性化设置。
